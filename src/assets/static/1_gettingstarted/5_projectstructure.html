Project Structure

We will go through important files only as part of this tutorials.

When you open your newly created poject in your IDE it will look like below:

    
[ProjectName]
    > e2e
    > node_modules
    > src
        > app
        > assets
        > environments
        > index.html
        > mail.ts
        > polyfills.ts
        ...a bunch of files
    package.json
    tsconfig.json
    tslint.json
    ...a bunch of files

e2e: This is required and used to create end to end automated tests.

node_modules: All third party libraries that we import to our application or our application have dependency will be available in this folder. This will be used during the build to create the bundles.

src: This is the folder where we will have actual source code of our application.

app: This is the folder we will be working most of the time during our development of the applications. App folder will have at least pne module file and one component files. We will look into these files in more details in following topics.

assets: This folder is used to save any static assets you have like image files, icons, etc. And the assets inside this folder will be available across the application.

environments: Configuration settings for different environments will be stored inside this folder. environment.prod.ts file will be referred in production mode and environment.ts file will be used for development mode.

index.html: This is the only page which is served for any angular application.

main.js: This is starting point of our application. This is used to provide the configuration for bootstrapping our application.

pollyfills.js: Angular uses the javascript features which are not available in the latest javascript versions supported by browsers. This file fills this gaps and provides all the javascript features angular needs so that angular app runs without any issues in all browsers.

package.json: In this file all our application dependencies of the libraries with respective versions will be listed out. devDepenencies needs during application development only and not required to run the application in production.

tsconfig.ts: This contains settings for TypeScript compiler.

tslint.json: This is Static analysis tool for TypeScript code same as Sonar Quebe you use for Java or .NET code to check code for readability, maintainability and functionality errors.
